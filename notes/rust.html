Two kinds of statements in Rust:

  • declaration statement (modules/functions/structs/enums/etc., or let bindings)
  • expression statement

function arguments must have their types declared

a function body is just a sequence of expressions, separated by semicolns. The return type of the function is the type of the last expression, unless there’s a trailing semicolon, which means the function returns (). You can also use the “return” keyword to return an expression.

Two kinds of comments

  • line comments (//)
  • doc comments (///, support markdown inside, used by rustdoc)

match statements: - exhaustiveness checking - can destructure the values in enum variants - also destructure structs and tuples

Remember, enums are scoped now.
